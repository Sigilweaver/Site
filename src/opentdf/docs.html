Rust quickstart

use opentdf::Reader;

let reader = Reader::open("my_bundle.d")?;
let calib = reader.calibration()?;   // TOF-to-m/z, scan-to-1/K0
let frame = reader.frame(1)?;
for peak in reader.decode_peaks(&frame)? {
    let mz = calib.tof_to_mz(peak.tof);
    let inv_k0 = calib.scan_to_inv_mobility(peak.scan);
    println!("mz={:.4} 1/K0={:.4} intensity={}", mz, inv_k0, peak.intensity);
}

Format layout

Each .d/ bundle contains:

  • analysis.tdf — SQLite database: frame metadata, scan count, calibration parameters, instrument settings.
  • analysis.tdf_bin — binary blob: one block per frame at the byte offset stored in Frames.TimsId. Block layout: [u32 block_size | u32 scan_count | payload].

Payload is compressed with either legacy LZF (codec 1) or zstd-with-transpose (codec 2), dispatched via TimsCompressionType in the SQLite schema.

Calibration

TOF-to-m/z and scan-to-1/K₀ both use the open opentims linear-in-sqrt(m/z) and linear models respectively, with coefficients read from the SQLite CalibrationInfo table.
